# learning/materials/foundations

The **maths** under the whole pipeline — the layer beneath both imaging physics
([../mri/](../mri/)) and the analysis stack ([../common/](../common/)). Working
fluency, not proofs; every topic tied to *where it appears in our code*.

## Read in this order
1. [curriculum.md](curriculum.md) — the plan + reference resources (Deisenroth etc.).
2. [linear-algebra.md](F1_linear-algebra.md) — vectors/matrices, the NIfTI affine, convolution as a linear map.
3. [calculus-optimization.md](F2_calculus-optimization.md) — gradients, backprop, gradient descent, reading a loss curve.
4. [probability-stats.md](F3_probability-stats.md) — softmax/cross-entropy, estimation, Bland-Altman, CV variance.

Made the same way as the rest (project circuit): grounded in
[../../../research/deep_dives/](../../../research/deep_dives/), taught, corrected,
written here; quiz on demand. The angle throughout: maths *as used in this pipeline*,
so it sticks for an engineer who learns by application.
